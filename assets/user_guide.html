1   Overview

Noobys GamePRo is a Windows desktop application that controls the GamePRo hardware — an Arduino Nano driving servo motors that physically press buttons on a Nintendo 3DS or Nintendo Switch. The software lets you:

  • Run automated scripts (shiny hunting, grinding, etc.).
  • Send manual button presses directly from the PC.
  • Watch a live webcam feed for on-screen event detection.
  • Calibrate each servo so every press is accurate for your setup.
  • Update firmware with one click, preserving calibration automatically.
Note: GamePRo hardware is sold separately at www.noobysgamepro.com. This guide covers the PC software only.

2   Installation & First Launch

Requirements

  • Windows 10 or 11 (64-bit)
  • USB cable to connect the GamePRo device
  • Webcam (required for vision-based scripts; optional otherwise)
  • Internet connection (for script downloads and firmware updates)

Steps

  1. Download GamePRo.exe from github.com/NoobysGamePRo/gamepro-app/releases/latest.
  2. Copy GamePRo.exe to a folder of your choice (e.g. C:\GamePRo\). No installer needed — it is a single portable executable.
  3. Double-click to launch. If Windows SmartScreen appears, click More info → Run anyway.
Tip: Keep GamePRo.exe in its own folder. A scripts/ folder is created alongside it when you download community scripts.

3   Connecting the Hardware

The status dot next to the Serial Port dropdown shows the connection state:

ColourMeaning
GreyNot connected / no port selected.
YellowConnecting…
GreenConnected and ready.
RedConnection failed.
  1. Plug the GamePRo into a USB port. Windows installs the driver automatically.
  2. The Serial Port dropdown auto-selects the first port on startup. If the dot is already green, you are connected.
  3. If the port is missing, click ↻ to rescan.
  4. Changing the dropdown to a different port reconnects automatically.
Important: Only one application can use a COM port at a time. Close the Arduino IDE Serial Monitor before launching GamePRo.

Webcam

Select your webcam from the Webcam dropdown (index 0 is normally the first camera). Click ↻ to rescan. Enable the feed with ● Preview: OFF — the button turns green when the feed is live. Vision scripts require the preview to be on.

4   Running Scripts

Scripts appear in the Scripts tree on the left, grouped by game generation (e.g. Gen 3 Frlg, Gen 4 Hgss).

  1. Confirm the dot is green (connected).
  2. Click a script name in the tree to select it.
  3. Click ▶  Run Script.
  4. Progress is shown in the Log panel below the tree.
  5. To abort at any time, click ■  Stop.
Vision scripts: Enable the webcam preview and point the camera at the console screen before running. If prompted, draw a detection region on the preview by clicking and dragging.

5   Manual Controls

The Manual Controls panel sits below the live preview. All buttons are disabled until the device is connected, and also while a script runs.

D-Pad arrows — double click Hold the direction continuously. Use Release All to return.
A (red)  B (amber)  X (blue)  Y (green) Tap the face button.

6   Live Preview & Calibration Regions

The 640×480 preview canvas shows the webcam feed in real time. Vision scripts use this feed to detect events on the console screen.

Drawing a detection region

  1. When a script prompts you, click and drag on the preview canvas to draw a rectangle over the area of interest (e.g. where the shiny star appears).
  2. Release the mouse to confirm the region.
  3. The region coordinates are passed to the running script for all subsequent checks.
  4. If you make a mistake, stop the script and rerun it to redraw the region.

7   Servo Calibration

Each servo has a press angle and a release angle, stored in the Arduino EEPROM. These survive firmware updates.

Click ⚙ Calibrate in the hardware row (requires connection).

GroupValues
D-Pad PositionsPress and release angles for Up, Down, Left, Right.
Face ButtonsPress and release angles for A, B, X, Y, Home/SR1, SR2, +.
Backoff & TimingSpeed and backoff distances for all servos.
  1. Open the dialog — values are read from the Arduino automatically.
  2. Adjust spinboxes (servo microseconds, roughly 500–2500).
  3. Click Write to Arduino to save to EEPROM.
  4. Click Read from Arduino to reload saved values and discard unsaved changes.
Warning: Incorrect values can cause servos to jam or miss buttons. Make small changes (5–10 units) and test with the Manual Controls after each save.

8   Firmware Updates

  1. Ensure the device is connected (green dot).
  2. Click ↑ 3DS Firmware or ↑ Switch Firmware.
  3. Confirm the dialog. The app will:
    1. Back up calibration from the Arduino.
    2. Download the latest .hex from GitHub Releases.
    3. Flash it using the bundled avrdude tool.
    4. Reconnect after the Arduino reboots.
    5. Restore the backed-up calibration.
Tip: The process takes about 10–15 seconds. The log shows each step. If the flash fails, check that no other application is using the COM port.
Note: If the existing firmware predates EEPROM calibration support (pre-v4.5 / Switch pre-v1.2), calibration backup is skipped and you will need to recalibrate manually after the update.

9   Script Builder

The Script Builder records button sequences with real timing, lets you mark detection regions, and exports a complete AI prompt. Paste the prompt into Claude, ChatGPT, or any AI assistant to generate a ready-to-run .py script.

  1. Click ⚒ Script Builder in the Scripts panel header.
  2. Enter a script name and description.
  3. Click Start Recording, then use the Manual Controls to perform the button sequence exactly as in gameplay.
  4. Click Stop Recording.
  5. Optionally click Add Detection Region to mark a screen area for the AI to generate detection code for.
  6. Click Export Prompt. The AI prompt is shown / copied to clipboard.
  7. Paste into an AI assistant. Save the generated .py file into scripts/ (or a subfolder).
  8. Click ↻ in the Scripts panel header to refresh the tree.

10   Downloading Community Scripts

  1. Click ↓ Get Scripts at the top of the Scripts panel.
  2. The app downloads the latest ZIP from github.com/NoobysGamePRo/gamepro-scripts and extracts .py files into the scripts/ folder next to GamePRo.exe.
  3. The log reports how many files were added or updated.
  4. The script tree refreshes automatically.
Tip: You can add your own scripts to any subfolder inside scripts/. The app discovers them automatically on startup and uses the subfolder name as the tree group heading.

11   Troubleshooting

SymptomLikely cause & fix
Red dot — Cannot connect to COMx Another program is using the port. Close it and click ↻ to reconnect.
No ports listed Driver not installed. Install the CH340 driver from the Arduino Nano supplier website.
Servos move but miss the button Open ⚙ Calibrate and adjust the press angle for the servo.
Hold direction releases immediately Firmware is older than v4.4. Use the firmware update buttons to update.
Script does not detect the shiny / event Ensure preview is on and camera is aimed at the console. Redraw the detection region.
Firmware flash fails Verify the correct COM port. Ensure the USB cable carries data. Unplug and replug the device.
Scripts folder empty after Get Scripts Check internet connection. The repository URL may not yet be active — check the log.

12   Advanced: Writing Your Own Scripts

All scripts inherit from BaseScript in scripts/base_script.py.

from scripts.base_script import BaseScript

class MyScript(BaseScript):
    NAME        = "My Script Name"
    DESCRIPTION = "What this script does"

    def run(self, controller, frame_grabber, stop_event, log,
            request_calibration):
        log("Script started")
        while not stop_event.is_set():
            controller.press_a()      # tap A
            controller.press_up()     # tap Up
            controller.hold_right()   # hold Right
            import time; time.sleep(0.5)
            controller.release_all()  # lift all servos

Key controller methods

MethodAction
press_a() / press_b() / press_x() / press_y() Tap a face button (blocks until servo returns).
press_up() / press_down() / press_left() / press_right() Tap a D-pad direction.
hold_up() / hold_down() / hold_left() / hold_right() Hold a direction until release_all().
soft_reset()Press Home / SR1.
release_all()Lift all servos.
read_light_value()Return light sensor reading (0–1020).

Vision detection pattern

import cv2, numpy as np

region = request_calibration(log)   # user draws rectangle; returns (x, y, w, h)
if region is None:
    return

x, y, w, h = region
while not stop_event.is_set():
    frame = frame_grabber.get_latest_frame()
    if frame is not None:
        crop = frame[y:y+h, x:x+w]
        brightness = np.mean(cv2.cvtColor(crop, cv2.COLOR_BGR2GRAY))
        if brightness > 180:   # threshold — adjust to taste
            log("Bright flash detected!")
            break

Placing your script

Save the .py file in scripts/ or any subfolder. The subfolder name (underscores → spaces, title-cased) becomes the group heading. No registration needed — the app discovers scripts automatically on startup.
